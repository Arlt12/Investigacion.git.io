Usabilidad

¿Qué es usabilidad web?

Hace referencia a los complementos que tiene una página web y estos permiten que sea fácil de usar para los usuarios.

Sus enfoques son el diseño web, presentaciones, información organizada y funcionamiento.

¿Cuál es la importancia de la usabilidad?

La usabilidad web hace un papel fundamental en la creación y optimización de sitios web.

Ya que da la facilidad con la que los usuarios pueden navegar y utilizar un sitio web para lograr sus objetivos.

10 principios de la usabilidad web

  • Visibilidad del estado del sistema
  • Consistencia entre el sistema y el mundo real
  • Control y libertad del usuario
  • Consistencia y estándares
  • Prevención de errores
  • Reconocimiento en lugar de recuerdo
  • Flexibilidad y eficiencia de uso
  • Diseño estético y minimalista
  • Ayude a los usuarios a reconocer, diagnosticar y recuperarse de errores
  • Ayuda y documentación

1.Visibilidad del estado del sistema

Hace referencia a que el programa debe informarle al usuario que es lo que está ocurriendo en el momento y no dejarlos pensando.

Ejemplos
1 Animaciones que muestran cuando un sitio se encuentra cargando.
2 Mensajes de confirmación al enviar un formulario.
3 mostrarle al usuario en que parte del sitio se encuentra.

2.Consistencia entre el sistema y el mundo real

Hace referencia a que el sistema debe de utilizar palabras familiares para el usuario y no palabras muy técnicas

Ejemplos
1 Adaptaciones de un mensaje a un idioma u otro.
2 El término 404 es un mensaje de “Página no encontrada”
3 El botón de “buscar” representado con una “lupa”

3.Control y libertad del usuario

Hace referencia a que siempre debe de haber un boton de salida en caso de que el usuario realice alguna accion por error.

Ejemplos
1 Quitar elementos de un carrito de compras.
2 Aplicar y borrar filtros al comprar productos.

4.Consistencia y estándares

Se debe seguir un estándar para no confundir a los usuarios,una vez creada la página la idea es no mover las acciones y demás a otro lugar.

Ejemplos
1 Mantener la consistencia interna.
2 Mantener las convenciones establecidas por la industria (“X” para cerrar, “Lupa” para buscar, entre otros.)”

5.Prevención de errores

Crear mensajes para prevenir errores diguitados por el usuario.

Ejemplos
1 Los placeholders en campos de texto, donde aclaran la acción que debemos realizar.
2 La comprobación de campos de un formulario en tiempo real.

6.Reconocimiento en lugar de recuerdo

Hace referencia el hacer visicibles todas las opciones posible para que así el usario minimice el uso de su memoria.

Ejemplos
1 Historial de reproducciones de YouTube.
2 muestran al usuario en que parte del sitio se encuentra.
2 Barra de progreso que muestra en que parte de un video te has quedado.

7.Flexibilidad y eficiencia de uso

Hace referencia a que los usuarios que son más frecuente deben de tner algún tipo de opcion "atajos" para llegar al lugar al que frecuentan.

Ejemplos
1 Configuración avanzada de tu cuenta de Spotify.
2 El reingreso a la app de Spotify te muestra las playlists que escuchas frecuentemente.

8.Diseño estético y minimalista

La interfaz no debe tener información innecesaria solo lo que el usario necesite y se debe conocer el tipo de contenido que desee el usuario.

Ejemplos
1 La página principal de Apple muestra solo los productos que desean destacar.
2 El buscador de Google solo muestra un campo de texto y un par de botones, no todos sus servicios.

9.Ayude a los usuarios a reconocer, diagnosticar y recuperarse de errores

AL tener un error los mensajes deben ser claros para el usuario, sin codigo de error.

También darles alguna solución para resolver el problema.

Ejemplos
1 Camuflar el código 404 o complementarlo con información adicional.
2 Cuando existen productos agotados, la web muestra un mensaje de cuando se reabastecerá el producto
3 Si inicias sesión en Facebook y colocas un dato erróneo, la app te explicará en que puedes estar fallando.

10.Ayuda y documentación

Se debe crear una documentación clara para que el usuario pueda realizar sus tareas deseadas sin necesidad de una explicación adicional.

Ejemplos
1 Chats de ayuda.
2 Tutorial o guías para herramientas web.
3 Sección de Preguntas frecuente.
4 Minitours en la primera visita del usuario a una interfaz.
